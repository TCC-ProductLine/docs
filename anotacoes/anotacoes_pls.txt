* O que é uma linha de produtos de software?
	A linha de produtos de software possui um enfoque na reutilização do código fonte. Mais do que uma nova tecnologia, é uma nova maneira de fazer negócios que pode render melhorias de ordem de magnitude em tempo de mercado, custo e qualidade. [2]

	Linha de produtos de software é um portfólio de sistemas baseados em software similares e produtos produzidos a partir de um conjunto compartilhado de ativos de software usando um meio comum de produção. [1]

	Uma linha de produtos de software é um conjunto de sistemas intensivos de software que compartilham um conjunto comum, gestão de recursos que satisfaçam as necessidades específicas de um determinado segmento de mercado.[2]

* Por que Linha de produtos de software é importante
	As linhas de produto pode ajudar as organizações a superar os problemas causados pela escassez de recursos. Organizações de todos os tipos e tamanhos descobriram que uma estratégia linha de produtos, quando habilmente executada, pode produzir muitos benefícios e, finalmente, dar às organizações uma vantagem competitiva [2]. Exemplo benefícios organizacionais incluem:
		1- Uma produtividade até 10 vezes maior;
		2- Aumento da Qualidade em até 10 vezes;
		3- Redução do custo em até 60%;
		4- Diminuição da necessidade de trabalho em até 87%;
		5- Diminuição do tempo de lançamento em até 98%
		6- Capacidade de entrar em novos mercados em meses, e não em anos.
	
* Linha de produtos de software e a nova geração de desenvolvimento de software
	Linhas de produtos de software estão emergindo como um paradigma de desenvolvimento viável e importante, permitindo que as empresas realizar melhorias de ordem de magnitude em tempo de mercado, custo, produtividade, qualidade e outros fatores de negócios. Linha de produto de software também pode permitir a entrada no mercado rápida e resposta flexível, e fornecer uma capacidade de customização em massa.[2]

A linha de produtos é um conjunto de produtos que, juntos, tratam de um determinado segmento de mercado ou cumprir uma missão particular.[2]

As linhas de produtos de software com base na semelhança entre produtos são um conceito relativamente novo que está emergindo rapidamente como um paradigma de desenvolvimento de software viável e importante. Flexibilidade do produto é o hino do mercado de software, e as linhas de produtos, para cumprir a promessa de sistemas sob medida é construido especificamente para as necessidades de clientes específicos ou grupos de clientes.[2]

A linha de produtos é bem sucedida, porque as semelhanças compartilhadas pelos produtos de software podem ser exploradas para obter economias de produção. Os produtos são construídos a partir de ativos comuns de uma forma prescrita.[2]

As organizações estão descobrindo que esta prática de conjuntos de construção de sistemas relacionados a bens comuns pode render notáveis melhorias na produtividade, tempo de mercado, a qualidade do produto e satisfação do cliente. Eles estão descobrindo que uma linha de produtos de software de forma eficiente pode saciar a fome atual de customização em massa. Organizações que adquirem, em oposição a construir, sistemas de software estão descobrindo que a colocação de um conjunto de sistemas relacionados, como uma linha de produtos comumente desenvolvido produz economias em tempo de entrega, custo, formação simplificada e aquisição simplificada.
